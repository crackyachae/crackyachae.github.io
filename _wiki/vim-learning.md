## 2020.03.15

+ Vim을 처음 접해봤다.
+ VS Code에 Vim 플러그인을 깔아서 가장 기본적인 것만 급하게 해결해서 작성하는중이다.

+ 배운 명령어
  + `i`로 입력모드 들어가서 `esc`로 나오기
  + `yy`로 문장복사 `p` 또는 `P`로 붙여넣기

## 2020.04.01

+ 공식사이트의 documentation, 학습을 도와주거나 잘 정리해둔 여러 사이트를 정리했고 [링크](0_Vim_Documentation_Intro.md) 학습 기반을 마련했다.

## 2020.04.02
+ 기본적인 기능부터 '조금씩', '확실히' 익혀보려고 하는 중.
+ Open Vim과 Vim Adventure로 이동과 복사, 붙여넣기, 삭제, 숫자로 반복 수행하는 정도만 익히기 시작.
+ 의식적으로 계속 쓰려고 노력하는게 중요한 것 같다.

## 2020.04.03
+ 어떤 기능이 있고, 그 기능에 해당되는 키 정도는 빠르게 떠올릴 정도는 되어야 뭘 쓰기라도 할 수 있을 것 같아서 Openvim과 Vim Genius를 꾸준히 하려고 하는 중.

## 2020.04.12
+ 사이트 Training 자체는 익숙해졌는데 여전히 실제로 사용할 때 떠오르는 정도가 늦어서 필요한 편집 단축키를 정리해서 바로 확인할 수 있는 곳에 붙여두었다.
+ 생각보다 이동하는 키가 훨씬 많았고 이전에 몰라서 불편했던 부분을 해결할 수 있는 키도 많았다.
+ 쓸 때마다 현재 하려는 편집을 할 때 가장 효과적인 키를 바로 확인할 수 있어서 편하다.

## 2020.05.13
+ Vim Genius는 메모 써붙인 이후로는 거의 하고 있지 않고 level 4까지 나오는 기능들은 대부분 손에 많이 익은 것 같다.
+ 메모 확인하지 않아도 바로 떠올려서 쓰고 있는 것 들은 여기에 표시하고 메모를 다른 기능으로 바꾸기 시작해봐야겠다.

+ 이동키들은 가장 처음부터 익히기 시작했는데 여전히 어렵다. 키 기능 자체는 다 외웠는데 정확히 원하는 위치로 이동하기 위해 써야할 키를 정하는 게 쉽지않다.
+ 문단 내 이동은 입력 상태에서 커서로 이동하는 경우가 많아서 가장 늦게 익숙해지는 것 같기도 하다.
+ 특히 문장 중간에 있는 단어를 찾아가는 건 아직도 vim을 사용하는게 훨씬 비효율적. 
+ 주로 사용하고 있는 방법은 
  + `(`나 `)`로 문장 첫/끝으로 이동한 뒤 커서로 이동.
  + `w`, `e`, `b` 연타. `2w`같이 앞에 숫자 넣어서 이동하는 건 단어 개수 가늠이 잘 안돼서 그런지 잘 사용하지 않게된다.
  + `f + 문자` 반복. 한글일 경우 택도 없음.
+ `n + G`를 이용해서 이동하는게 가장 익숙하고 `$`랑 `^`도 가장 많이 익숙하게 쓰는데 키 위치가 손에 안익어서 키보드를 확인하고 눌러야하는게 아쉽다.

이 기능들은 무리 없이 사용하는 중.

| key    | 의미                 | key    | 의미              |
| ------ | ------------------ | ------ | --------------- |
| `gg`   | 글 처음으로 이동          | `G`    | 글 마지막으로 이동*     |
| `^`    | 문단 처음으로 이동         | `$`    | 문단 마지막으로 이동     |
| `(`    | 이전 문장으로 이동         | `)`    | 다음 문장으로 이동      |
| `b`    | 이전 단어로 이동          | `w`    | 다음 단어로 이동       |
| `F+문자` | 이전에 처음 발견되는 문자로 이동 | `f+문자` | 다음번 발견되는 문자로 이동 |

+ \* `n+G` n번째 문장으로 이동

+ 입력모드로 들어가고 나오는게 가장 빨리 익숙해진 것 같다. 처음엔 무조건 `i`만 썼는데 `a`의 존재를 써 붙인 이후로 둘 다 자유롭게 쓰고 있는 편.
+ 손이 잘 안 닿는 이동키 대신 이동한 뒤 입력으로 전환되는 키들을 점차 늘리면서 익숙해지는 중이다.

현재 익숙해진 기능들은 이정도.

| key | 의미                  | key | 의미                   |
| --- | ------------------- | --- | -------------------- |
| `i` | 현재 커서에 앞에 입력        | `a` | 현재 커서 뒤에 입력          |
| `I` | 현재 문장 가장 앞에서 입력     | `A` | 현재 문장 가장 뒤에서 입력      |
| `O` | 현재 문장 위에 빈줄을 만들어 입력 | `o` | 현재 문장 아래에 빈줄을 만들어 입력 |

+ 편집 기능은 기능 자체는 그냥 사용할 수 있는데 영역 선택이 어렵다. 그래서 연속명령으로 문장 단위를 편집하는 걸 주로 사용한다.

| key | 의미                |
| --- | ----------------- |
| `y` | 복사                |
| `d` | 삭제                |
| `c` | 변경(잘라내고 입력모드로 전환) |

+ `+l`: 한 문자에 적용
+ `+w`: 단어에 적용
+ 대문자: 현재 커서에서부터 라인 끝까지 적용
+ 연속명령: 문장 전체에 적용

| key | 의미         | key | 의미         |
| --- | ---------- | --- | ---------- |
| `P` | 커서 앞에 붙여넣기 | `p` | 커서 뒤에 붙여넣기 |


## 2020.05.17
+ Vim Genius 모든 단계를 끝냈다. level 5-7 명령어 대부분이 이미 많이 쓰고 있던거라 한 번에 끝냈고 앞으로는 자주 안 봐도 될 것 같다.
+ 이동키를 더 알아봐야겠다. 화면에 벗어나는 순간 마우스를 사용하게 되어서 최대한 다양하게 기억하고 이동해보면서 적절한 이동방식을 찾아야겠다.
